Technical Documentation

Overview of the system architecture and vanilla implementation.

Custom Styling without Frameworks

This project uses a fully custom CSS architecture (Vanilla CSS) designed to be lightweight, fast, and completely independent of heavy frameworks like Bootstrap.

Styling Architecture

  • CSS Variables: All theme colors are mapped in styles.css under the :root pseudo-class.
  • CSS Grid & Flexbox: Layouts utilize native CSS Grid (for gallery and event cards) and Flexbox (for navigation and forms).
  • Responsive Design: Media queries adapt the layout for mobile users without relying on pre-defined column classes.
Customizing Colors

To change the primary theme color, open styles.css and edit the --primary and --primary-hover variables at the top of the file.

Vanilla JavaScript Integration

The logic is built purely on modern ES6+ vanilla JavaScript.

  • Modals: Custom modals are toggled by applying the .show class via JS classList.add().
  • Data Fetching: Uses native fetch() API with async/await for mock API integrations.
  • DOM Updates: Utilizes document.createElement and template literals for fast rendering.
